## Usage Guidelines

### Color
✅ **Do:** Use semantic tokens exclusively in components. Never hardcode hex values.
❌ **Don't:** Use primitive tokens directly in components — primitives are the source layer only.

### Typography
✅ **Do:** Use the display face for headings ≥ heading-2. Use the UI face for all body, label, and UI text.
❌ **Don't:** Mix face assignments arbitrarily. The pairing logic must be applied consistently.

### Dark mode
✅ **Do:** Switch themes via Figma variable mode. Verify each component independently in dark — never assume light mappings are correct.
❌ **Don't:** Treat dark mode as an inversion. `surface-page` in dark is not `surface-page` in light with a `filter: invert()`.

### Spacing
✅ **Do:** All padding, margin, and gap values must use a space token. 4px grid only.
❌ **Don't:** Use arbitrary values (e.g. 13px, 22px). If a spacing need doesn't fit the scale, question the layout decision first.

### Touch targets
✅ **Do:** All interactive elements must be ≥ 44×44px on mobile. Add padding if the visual element is smaller.
❌ **Don't:** Use desktop-only sizing for components that appear on mobile screens.
